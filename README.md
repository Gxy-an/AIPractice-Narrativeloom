## 科学问题 1：人格功能分化对人机协同叙事多样性与连贯性的影响

NarrativeLoom采用类型化人格（Genre-based Persona）提升创意，但笔者使用时个人认为这样的创意突变有些跳脱，考虑进一步探究功能化人格是否能更好平衡创意多样性与叙事连贯性。

### **1.1** **基本设想**

功能化人格是基于叙事创作的核心功能和任务模块设计的AI协作人格，不按文学题材（奇幻、科幻、悬疑）划分，而是按故事创作的具体环节、叙事要素、创作任务分工，每个人格只专注解决1个创作方面，为故事节拍提供专业化、定向化的创意支持。本质是创作功能的纵向分化，而非题材风格的横向拓展。

### **1.2** **可行性分析与研究价值**

在叙事上，故事由设定、人物、情节、氛围、细节等核心功能模块构成，功能分化符合叙事创作的客观规律；在认知负荷上，单一功能人格降低用户筛选成本，避免多题材切换带来的认知混乱；在人机协同分工上，AI专注专业化功能，用户聚焦整体创意决策，类似筑梦师角色，回归创意导演定位。定向化功能变异可能比泛题材变异更精准，在保留盲变异的创意性，提升选择性保留的效率。

​    在NarrativeLoom的题材泛化变异基础上，探索创意支持的功能化垂直分工，回答**“AI人格如何按创作任务分工，才能平衡创意与连贯性”**的问题，为创意AI提供可复用的功能化设计。

### **1.3** **具体的功能化人格设计**

基于故事节拍的地点、时间、人物、事件三大核心要素，拟设计8个核心功能化人格，覆盖创作全流程。此处类比《盗梦空间》Cobb团队，每个成员都有明确的分工。

| 人物     | 角色            | 任务                                       |
| -------- | --------------- | ------------------------------------------ |
| Dom Cobb | 盗梦师Extractor | 统筹全局，执行植入，确保各层梦境协调       |
| Ariadne  | 建筑师Architect | 设计和稳定梦境空间物理结构                 |
| Arthur   | 前哨Point Man   | 负责任务细节规划、信息收集与现场执行       |
| Eames    | 伪装者Forger    | 伪装身份，引导目标，植入想法               |
| Yusuf    | 药剂师Chemist   | 研发镇静剂，确保团队进入稳定的多层梦境     |
| Saito    | 资助者Financier | 提供任务资金与资源，清除障碍，监督任务执行 |

具体功能化人格设计想法如下：

1. **设定构建师**：专注故事的时空、场景、世界规则设计（解决场景单薄问题）
2. **人物塑造师**：专注角色性格、动机、关系、行为逻辑设计（解决人物扁平问题）
3. **剧情逻辑师**：专注事件因果、冲突、反转、节奏设计（解决剧情漏洞问题）
4. **氛围渲染师**：专注情绪、感官、氛围感描写（解决叙事干瘪问题）
5. **对话设计师**：专注角色台词、语气、互动设计（解决对话生硬问题）
6. **细节填充师**：专注道具、伏笔、细节补充（解决故事粗糙问题）
7. **冲突设计师**：专注核心矛盾、阻碍、高潮的整体波动设计（解决剧情平淡问题）
8. **连贯性校验师**：专注剧情前后逻辑、人物状态一致性（辅助RAG控制器）

### **1.4** **人机共创流程优化**

每个功能化人格只生成对应功能的节拍片段/方面/要素（而非完整节拍），用户可组合不同功能片段拼接成完整节拍，将多题材并行生成，改为多功能并行生成。连贯性控制器优先校验功能人格生成内容的逻辑一致性，以减少剧情冲突。

## 科学问题 2：反馈模式对人机协同叙事迭代质量的影响

NarrativeLoom已通过多人格盲变异+人类选择性保留实现创意突破，但AI 生成全程为 “黑箱”，系统仅输出故事节拍/文本结果，不向用户解释人格创作思路、节拍设计依据、叙事逻辑关联。笔者使用时感觉可能存在以下问题（不确定表述是否恰当）：

1. 用户无法理解AI创作逻辑，人机认知对齐度低；
2. 迭代时反复修改，效率较低；
3. 用户难以吸收AI的叙事技巧，故事精致性高但人物塑造深度难以提升。

### **2.1** **基本设想**

基于NarrativeLoom的BVSR协同创作框架，聚焦AI可解释性缺失与人机认知不对齐的核心痛点，拟比较**结果反馈与过程反馈对人机协作结果的叙事质量（故事精致性、人物塑造深度）和人类创作体验（迭代修改效率、创作理解度、创意启发度）的影响**。重在提升AI创意支持的可解释性，强化人机协同的认知对齐，完善人机协同叙事的认知交互机制。

其中，结果反馈（对照组）为AI仅输出标准化内容，无任何附加解释，沿用NarrativeLoom原生输出模式；过程反馈（实验组）为AI输出生成内容和结构化创作解释，拟包含3部分：

1. 人格创作目标；
2. 节拍设计依据（与前文的逻辑关联）；
3. 叙事技巧说明（环境氛围/人物形象/冲突设计思路等）。

### **2.2** **核心困难**

为比较两种反馈模式，需解决下列研究问题：

1. 如何结构化输出 AI 创作思路，既实现可解释性，又不增加用户认知负荷；
2. 如何量化人机认知对齐、人物塑造深度等抽象指标；
3. 控制无关变量（人格、节拍、模型），精准分离反馈模式的独立效应。

### **2.3** **具体的评估指标**

评估指标包含客观因变量与主观因变量两部分。

| 因变量                 | 评估指标       | 具体内容                                   |
| ---------------------- | -------------- | ------------------------------------------ |
| 叙事质量  （客观）     | 迭代修改效率   | 单节拍修改耗时、总修改次数                 |
|                        | 故事精致性得分 | TTCW创意量表评分                           |
|                        | 人物塑造深度   | 人物动机完整性、情感复杂性、行为逻辑合理性 |
|                        | 叙事连贯性     | 剧情冲突错误数量、逻辑漏洞数               |
| 人类创作体验  （主观） | 创作理解度     | 用户对AI生成逻辑、叙事意图的理解程度       |
|                        | 创意启发度     | AI反馈对用户创意拓展、思路启发的效果       |
|                        | 认知负荷       | 完成创作的心理负担、创作过程中的阅读压力   |
|                        | 协同满意度     | 对人机协作过程的满意度                     |

## 科学问题 3：多人格生成顺序对人机协同叙事故事多样性与连贯性的影响

探索多人格AI的协作时序对叙事创作的影响，如随机并行生成vs按叙事逻辑递进生成，完善分布式创意理论。

## 科学问题 4：创作阶段适配的人格动态切换对新手创作者创意提升的影响

针对新手获益更大的研究结论，优化新手创意支架设计，比较固定人格池vs按创作阶段（构思/发展/高潮）动态建议切换人格的模式对新手创意启发感、创作自信心、TTCW创意得分、故事完成度、场景丰富度等的影响。
